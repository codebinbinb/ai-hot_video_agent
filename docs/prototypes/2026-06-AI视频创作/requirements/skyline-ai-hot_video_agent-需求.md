## 需求背景

短视频已成为「获客 / 营销 / 私域 / 起号 / 运营」的核心载体，但内容生产存在三大痛点：① 选题与文案依赖人工对标爆款，效率低；② 出镜口播/达人带货需要真人拍摄，成本高；③ 多平台分发需重复编辑标题/封面/文案并逐个上传，繁琐易错。

本项目打造一个 **Agent + LLM 驱动的「IP 搭建 AI 助理」**：以智能体编排串联"对标拆解 → 文案仿写 → 配图/配音 → 成片 → 多平台分发"全流程，让运营者**低成本、批量化**产出并分发短视频内容。MVP 以 **B/S Web 端**形态、**云端 API**（无需 GPU、无需服务器）落地，便于本地/内网快速使用，后续可扩展桌面客户端与 Web 后台管理端。

## 目标用户

| 用户角色 | 特征 | 使用场景 |
|----------|------|----------|
| IP 运营 / 起号者 | 需要批量起号、稳定产出，懂运营不一定懂剪辑 | 对标爆款拆解 → 仿写 → 批量成片 → 多平台分发 |
| 达人带货 / 商家 | 有商品与达人形象，需大量带货口播视频 | 用达人参考图 + 脚本生成口播带货短视频并发布 |
| 私域 / 营销人员 | 需面向不同平台风格化内容做获客与私域引流 | 一份内容按抖音/小红书/视频号/快手风格分发 |

> MVP 范围：**单角色用户**（不区分权限角色），**素材库共享**。

## 功能描述

一个由 **Agent 编排**的短视频创作与分发助理，统一通过 **API 中转端点**（OpenAI 兼容）调用云端 AI 能力，串联从灵感到分发的完整链路。

### 核心功能

1. **M1 文案 → 配图（文生图）**：输入文案，LLM 进行润色/扩写与分镜，调用文生图模型生成竖屏（9:16）配图；配图数量按分镜数确定，用户可上传**最多 4 张参考图**（风格/达人参考）。
2. **M2 短视频生成**（两种形态）：
   - **图文字幕视频**：配图 + 字幕 +（用户上传音频 **或** AI 配音 TTS）+ 背景音乐，由 FFmpeg 合成；
   - **达人口播视频**：达人参考图 + 分镜提示词 → 云端**图生视频**（Veo / Seedance）生成口播带货片段。
3. **M3 视频拆解**：导入视频链接（抖音 / TikTok 等）→ 下载 → ASR 转写口播 → LLM 输出 **分平台风格仿写文案 + 分镜脚本（时间轴/画面描述/口播文案）+ 文生图提示词**。
4. **M4 一键发布**：自动生成 **标题 + 正文 + 标签 + 封面（多张多样式可选）**，发布到 **抖音 / 快手 / 小红书 / 视频号 + TikTok**（浏览器自动化）。
5. **M5 素材库（共享）**：成片视频、拆解视频、图片、音频、文案、提示词、脚本均可**一键或手动入库**；支持**手动上传维护**、标签 / 分类 / 检索。
6. **M0 Agent 编排（贯穿）**：以智能体编排各步骤为可观测的任务流水线，支持进度反馈、失败重试与人工介入确认。

### 功能边界

- ✅ 包含：文生图、图文字幕成片、达人口播（云端图生视频）、链接拆解、多平台一键发布、共享素材库、Agent 编排、统一云端 AI 接入。
- ✅ 包含：竖屏 1080×1920、≤30s 短视频；分平台语言风格与多语言仿写；标题/封面自动生成。
- ❌ 不包含（MVP）：本地 GPU 数字人/对口型大模型（HeyGem 等）；长视频剪辑/复杂时间线编辑（后续接剪辑工具）；多角色权限体系与多租户隔离；服务器/云端部署运维；后台管理端与桌面客户端。

## 用户故事

- 作为 **IP 运营**，我希望**粘贴一条对标视频链接就能自动拆出脚本、分镜和仿写文案**，以便**快速复用爆款结构产出新内容**。
- 作为 **IP 运营**，我希望**输入文案就能自动配图并合成带字幕的竖屏短视频**，以便**无需剪辑也能批量成片**。
- 作为 **达人带货商家**，我希望**上传达人参考图并给出脚本就能生成口播带货视频**，以便**省去真人反复拍摄的成本**。
- 作为 **运营人员**，我希望**同一条内容能按抖音/小红书/视频号/快手的风格自动改写**，以便**一次创作多平台分发**。
- 作为 **运营人员**，我希望**一键把成片发布到多个平台并自动填好标题/正文/标签/封面**，以便**省去逐平台手动上传**。
- 作为 **用户**，我希望**把生成的视频连同文案/图片/提示词一并存入共享素材库并可检索复用**，以便**沉淀与复用创作资产**。
- 作为 **用户**，我希望**配音既能上传自有音频也能用 AI 合成**，以便**灵活适配不同内容**。

## 业务规则

| 编号 | 规则 | 说明 |
|------|------|------|
| BR01 | 视频默认规格 | 竖屏 **1080×1920**、时长 **≤30s**、帧率保证流畅（建议 25/30fps） |
| BR02 | 配图数量 | 按用户提示词的**分镜头数量**确定；用户上传参考图**最多 4 张** |
| BR03 | 配音来源 | **上传音频** 或 **AI 配音(TTS)** 二选一，单条视频只用一种 |
| BR04 | 视频形态 | 支持「图文字幕」与「达人口播」两类；达人口播经云端图生视频实现 |
| BR05 | 仿写风格 | 支持分平台语言风格（抖音/小红书/视频号/快手等）与多语言输出 |
| BR06 | 发布平台 | 国内：抖音/快手/小红书/视频号；海外：TikTok；均经浏览器自动化 |
| BR07 | 发布内容 | 发布需携带标题、正文、标签、封面；封面自动生成多张多样式供选 |
| BR08 | AI 接入 | 统一走中转端点 `https://oneapi.itniotech.cn/v1`（OpenAI 兼容），模型可配置/替换 |
| BR09 | 凭据安全 | API 令牌与各平台登录态(Cookie)存于 **gitignored** 配置/加密存储，**严禁入库** |
| BR10 | 合规归责 | 提供免责声明：素材版权与内容合规由使用方自行承担，平台仅提供技术工具 |
| BR11 | 素材库 | 单一**共享**库；入库对象含视频/图片/音频/文案/提示词/脚本，支持标签/分类/检索 |
| BR12 | 任务编排 | 生成类任务异步执行，提供进度与失败重试；关键步骤可人工确认 |

## 验收标准

- [ ] AC01：输入一段文案，可生成符合分镜数量的竖屏配图，并可上传最多 4 张参考图。
- [ ] AC02：选择「图文字幕」，上传音频或选择 TTS 配音后，可合成带字幕+BGM 的 1080×1920、≤30s 视频。
- [ ] AC03：选择「达人口播」，上传达人参考图并给出脚本提示词后，可经云端图生视频生成口播片段。
- [ ] AC04：粘贴一条平台视频链接，可成功下载并 ASR 转写，输出仿写文案 + 分镜脚本 + 文生图提示词。
- [ ] AC05：仿写可按所选平台风格与语言输出不同版本。
- [ ] AC06：发布前可自动生成标题与多张候选封面，并可编辑标题/正文/标签后选定封面。
- [ ] AC07：可将成片一键发布到所选的国内平台与 TikTok（账号已登录态有效时）。
- [ ] AC08：可将成片/拆解视频连同文案/图片/提示词一键入库，并通过标签/关键词检索到。
- [ ] AC09：素材库支持手动上传与维护（增删改、打标签）。
- [ ] AC10：所有 AI 调用经中转端点完成；更换模型 ID 后功能仍可用（可配置）。
- [ ] AC11：API 令牌/登录态均不出现在仓库中（`.gitignore` 生效）。
- [ ] AC12：生成类任务有可见进度，失败可重试且有清晰错误提示。

## 边界与异常

| 场景 | 触发条件 | 处理方式 |
|------|----------|----------|
| 链接解析失败 | 链接失效/不支持/需登录 | 提示不支持或失效，引导改用上传或更换链接 |
| 视频下载失败 | 平台限制/网络异常 | 重试 N 次后报错，记录日志并提示原因 |
| AI 接口错误/限流 | 端点 4xx/5xx/超时/额度不足 | 退避重试；持续失败则反馈具体错误（含模型 ID/额度提示） |
| 模型不可用 | 端点缺少所需模型（如视频模型） | 明确提示缺失项并建议联系产品方/更换模型 |
| 对口型精度不足 | 达人口播需与指定配音严格对口型 | 提示当前图生视频不保证逐字对口型，建议评估专用对口型 API（开发期验证） |
| 配音/音频格式异常 | 上传音频格式不支持/时长超限 | 校验并提示支持的格式与时长（与 ≤30s 对齐） |
| 发布登录态失效 | 平台 Cookie 过期/被风控 | 提示重新登录该平台账号；失败不影响其他平台 |
| 发布被平台拦截 | 触发平台风控/内容审核 | 标记该平台发布失败并保留草稿，提示人工处理 |
| 素材检索为空 | 无匹配标签/关键词 | 返回空态并给出筛选建议 |
| 任务并发冲突 | 同时大量生成 | 任务排队，提示当前进度与预计等待 |

## 非功能需求

| 类型 | 要求 |
|------|------|
| 架构 | B/S；MVP 本地/内网运行，**无需服务器、无需 GPU**；后端 FastAPI + 前端 React/TypeScript |
| AI 接入 | 统一 OpenAI 兼容中转端点；Base URL=`https://oneapi.itniotech.cn/v1`；模型可配置 |
| 性能 | 交互响应及时；生成类长任务异步化并展示进度；合成/下载有超时与重试 |
| 安全 | 令牌、平台登录态加密/隔离存储且不入库；最小权限；本地数据隐私 |
| 兼容 | 现代浏览器（Chrome 等）；输出竖屏 1080×1920、≤30s，符合抖音等平台规范 |
| 可维护 | AI 模型/平台适配以可插拔方式实现，便于替换与扩展 |
| 合规 | 内置免责声明；下载与发布的版权/平台 ToS 风险由使用方承担 |
| 可扩展 | 预留多角色权限、后台管理端、桌面客户端、长视频剪辑接入的演进空间 |
