## 产品定位

打造一个 **Agent + LLM 驱动的「IP 搭建 AI 助理」**，围绕 **获客 / 营销 / 私域 / 起号 / 运营** 场景，帮助用户低成本批量产出并分发短视频内容。

- **不是单纯的 SaaS 工具**，而是强调 **Agent 编排能力** 的智能助理；
- **形态演进**：MVP 为 **Web 端**；后续可扩展 **桌面客户端** 与 **Web 后台管理端**；
- **架构**：B/S，MVP 本地/内网运行、无需服务器与 GPU（全部 AI 走云端 API 中转）。

## 澄清问答

### 一、范围与定位

#### Q1: MVP 包含哪些核心功能模块？
**A**: 4 大模块全要 —— ①文案生成图片 ②音频生成短视频 ③视频拆解为脚本/分镜/提示词 ④一键发布；**素材库**为支撑功能默认包含。**数字人口播**纳入 MVP（见 Q3 修订）。

#### Q2: 产品形态与目标用户？
**A**: B/S Web 端、Agent + LLM 驱动的 IP 搭建 AI 助理；**MVP 仅单角色用户，素材库共享**；后续可扩展客户端与后台管理端。

#### Q3: 是否纳入数字人口播？（重要修订）
**A**: **纳入 MVP**。因有"达人带货"场景，采用**云端图生视频**（达人参考图 + 提示词控制），核心是 agent 编排与 LLM 能力，而非本地 GPU 数字人。

### 二、技术路线

#### Q4: AI 能力用云端还是本地？
**A**: **云端 API 优先**，统一走 API 中转 `https://oneapi.itniotech.cn/v1`（OpenAI 兼容）。无需 GPU。

#### Q5: 技术栈？
**A**: 沿用本项目脚手架 **FastAPI（后端）+ React/TypeScript（前端）**。

#### Q6: AI 选型谁来定？
**A**: 由 AI 推荐（已基于中转端点 234 个可用模型给出，见"澄清结论 · AI 选型"）。

### 三、功能细节

#### Q7: 视频拆解输出深度？
**A**: **完整脚本 + 提示词**（分镜：时间轴/画面描述/口播文案 + 文生图提示词，可驱动二次创作）。

#### Q8: 图文/口播视频的配音来源？
**A**: **用户上传音频 或 AI 配音(TTS) 二选一**。

#### Q9: 标题/封面是否 AI 自动生成？
**A**: **要**，标题 + 封面均自动生成（多张封面 + 多样式供选）。

#### Q10: 视频规格？
**A**: 默认 **1080×1920 竖屏**（抖音竖屏格式），**时长 ≤30s**，帧率保证流畅（建议 25/30fps）；长视频剪辑能力**后续**接剪辑工具。

#### Q11: 配图/参考图数量？
**A**: 每条视频配图数量**按用户提示词的分镜头**确定；用户**最多上传 4 张参考图**（含达人参考图）。

#### Q12: 是否支持分平台语言风格？
**A**: **支持**，纳入不同社媒平台（抖音/小红书/视频号/快手 等）的语言风格（及多语言）。

### 四、发布方式

#### Q13: 一键发布平台与实现？
**A**: 国内（抖音/快手/小红书/视频号）+ **TikTok**。国内建议复用 `social-auto-upload`（浏览器自动化）；**TikTok 用浏览器自动化（Chrome CDP 等），开发后再验证**。发布自动带标题/正文/标签/封面。

## 澄清结论

### 1. MVP 功能范围

1. **文案 → 配图**：输入文案，LLM 润色/分镜，文生图生成竖屏配图。
2. **音频/文案 → 短视频**（两类）：
   - **图文字幕视频**：配图 + 字幕 +（上传音频或 TTS）+ BGM，FFmpeg 合成；
   - **数字人/达人口播视频**：达人参考图 + 提示词 → 云端图生视频生成口播片段。
3. **视频拆解**：导入链接（抖音/TikTok 等）→ 下载 → ASR 转写 → LLM 输出分平台仿写文案 + 分镜脚本 + 文生图提示词。
4. **一键发布**：自动生成标题/正文/标签/封面，发布到抖音/快手/小红书/视频号 + TikTok。
5. **素材库（共享）**：视频/图片/音频/文案/提示词/脚本一键或手动入库，支持上传维护、标签/分类/检索。

### 2. 创作流水线（Agent 编排）

```text
文案输入/链接拆解 → 文案润色·分平台仿写(LLM) → 配图(文生图) →
  ├─ 图文字幕：配音(上传/TTS) + 字幕(ASR) + BGM → FFmpeg 合成
  └─ 达人口播：达人参考图 + 分镜提示词 → 图生视频(Veo/Seedance)
→ 标题/封面自动生成 → 一键发布(多平台) → 入素材库
```

### 3. AI 选型（已验证中转端点可用，可替换）

| 能力 | 推荐模型（端点内 ID） | 备选 |
|------|----------------------|------|
| LLM / Agent 编排 | `deepseek-v3.2`、`qwen3-max` | `gpt-5.x`、`claude-opus-4.x`、`gemini-2.5-pro`、`kimi-k2`、`glm-4.6` |
| 多模态视觉理解（拆解画面） | `qwen/qwen3-vl-32b-instruct` | `gemini-2.5-pro`、`gpt-4o` |
| 文生图（配图/封面） | `Doubao-Seedream-3.0-t2i`（即梦） | `gpt-image-1`、`google/gemini-3-pro-image-preview` |
| 图生视频 / 达人口播 | `google/veo-3.1`（或 `veo-3.1-fast`） | `doubao-seedance-2-0-260128`、`Doubao-video-Seaweed` |
| 语音合成 TTS | `tts-1` | — |
| 语音转写 ASR（字幕/拆解） | `whisper-1` | `gpt-4o-transcribe` |
| 向量检索（素材库） | `text-embedding-3-large` | `bge-m3` |

**API 配置**：Base URL = `https://oneapi.itniotech.cn/v1`；令牌由产品方单独提供，**存于 gitignored 环境变量，严禁入库**。

### 4. 发布方案

- **国内**：复用开源 `social-auto-upload`（Playwright 浏览器自动化）。
- **TikTok**：浏览器自动化（Chrome CDP 等），**开发后验证**。
- 风险：浏览器自动化模拟人工，存在平台风控/ToS 风险，需安全保存账号登录态(Cookie)。

### 5. 合规

参考竞品**免责声明**模式：素材版权与使用合规由使用方自行承担，平台仅提供技术工具；下载第三方视频仅用于学习/仿写参考。

## 待确认 / 开发期验证项

- [ ] **数字人对口型精度**：云端图生视频（Veo/Seedance）可生成达人口播片段，但**与"指定上传配音"精确对口型(lip-sync)**通常需专用对口型能力；开发期需验证 Veo 自带音轨/口型是否满足达人带货需求，否则评估接入专用对口型 API。
- [ ] **TikTok 浏览器自动化**：Chrome CDP 方案的稳定性与风控，开发后实测。
- [ ] **中转端点模型可用性**：文生图/视频模型（Seedream/Veo/Seedance）的实际调用配额与计费，开发期联调确认；如缺失及时反馈。
- [ ] **API 令牌安全**：项目结束后建议轮换该令牌（已在对话中明文出现）。
- [ ] **长视频剪辑**：后续接入剪辑工具的范围与选型。
- [ ] **GitHub 仓库**：在 codebinbinb 下新建仓库 `skyline-ai-hot_video_agent` 并推送（需认证配合）。
